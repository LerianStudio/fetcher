2. Current Architecture (Working)

Manager Flow (HTTP API — Fiber:4006) Client → [JWT with tenantId] → Manager API (Fiber:4006) → Auth Middleware → TenantMiddleware (tmmiddleware.NewTenantMiddleware + WithMB) → tmmongo.Manager resolves tenant DB → Handler → MongoDB (per-tenant) → valkey.GetKeyContext → Redis (tenant-prefixed keys) → Publish to RabbitMQ (X-Tenant-ID header + per-tenant vhost via tmrabbitmq.Manager) Worker Flow (RabbitMQ Consumer) Worker → tmconsumer.MultiTenantConsumer (per-tenant vhost consumers, on-demand) → extractTenantIDFromHeaders → Resolve tenant MongoDB → tms3.GetObjectStorageKeyForTenant → S3/SeaweedFS (tenant-prefixed keys) → Publisher: tmrabbitmq.Manager (per-tenant vhost publishing)
